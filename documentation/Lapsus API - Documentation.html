    Lapsus API

    General notes

    Esta referência da API busca complementar o sistema e a interface desenvolvida, porém muitos “endpoints” são maleáveis quanto ao que precisa ou não ser enviado para o processamento da API. Sendo assim, de forma complementar consulte os arquivos validadores das requisições recebidas pelas rotas localizado no diretório “validators” dentro da pasta source. As validações são realizadas por meio da biblioteca javascript Joi, em caso de duvida consulte sua documentação, disponível aqui, além disso algumas validações são realizadas por meio de Expressões Regulares (RegEx).

    Todas as respostas das requisções possuem um atributo “success” de valor booleano, sendo verdadeiro em caso de sucesso e falso em caso de erro, para esse segundo caso, o retorno sempre será acompanhado de um atributo “message” descrevendo o por que do ocorrido.

    IMPORTANTE: Caso não deseje que um atributo seja interpretado pela API não o adicione no JSON da requisição, nem mesmo como vazio, caso contrário ele será interpretado e sua requisição será bloqueada e retornará com um HTTP status 400.

    OBSERVAÇÃO: Todos os dados de usuários apresentados tem a intenção de serem fictícios e não buscam relação com o mundo real.
